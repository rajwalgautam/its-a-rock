# v1.5.1

The planner shows where your body is, gives you finer control over the markers, and stops bumping holds by accident.

## What's new

**Route planner**
- **Current-stance indicator** — the up-to-four markers that make up your current body position (each limb's latest placement) now wear a bright amber ring, so you can pick out the current left hand / right hand / left foot / right foot at a glance.
- **Cleaner markers** — markers stay small and unnumbered so they barely cover the holds; a move's sequence number now appears only on the marker you tap.
- **Floating holds restyled** — a hold you mark "floating" now keeps its color and gets a dashed black outline instead of being greyed out, so it stays easy to see.
- **Marker opacity** — a new **Opacity** control sits next to **Bubble size**. Turn the bubbles and badges more transparent to see the holds underneath, or back to solid.
- **Deliberate dragging** — a marker now only moves after a short press-and-hold, so tapping to select or panning the photo no longer nudges holds out of place. A gentle tap when you pick one up confirms it.
- **Floating moved to double-tap** — because holding now lifts a marker to drag it, toggling a hold "floating" is now a **double-tap**.

## Under the hood

- New pure `currentStanceKeys` helper (unit-tested) returns each limb's latest move key; the planner flags those markers `current` and `LimbMarker` draws an amber stance ring that scales with bubble size. Edit markers now render compact by default, with the sequence badge shown only for the selected marker.
- Floating markers no longer reduce opacity; instead the dot uses a dashed black border (`borderStyle: 'dashed'`), keeping the limb color.
- Marker gestures reworked: drag is a `Pan` with `activateAfterLongPress` (light lift haptic), floating toggles on a double-tap, and single-tap still selects — composed so double-tap wins over single-tap.
- New discrete `BUBBLE_OPACITIES` scale with `nearest`/`step` helpers (unit-tested) and a persisted `bubbleOpacity` setting; the edit bar's size control was generalized into a shared stepper reused for both size and opacity. Opacity applies to the marker (dot, badge, ring).
