# v0.4.1

A patch release focused on improving the photo picker and date picker experience in the climb form.

## What's fixed

**Photo picker improvements**
- Photo taken via the floating action button (FAB) now displays as a preview in the form instead of showing empty camera/library buttons
- Photo picker modal now closes when tapping outside the menu, improving the interaction flow
- Photos taken with the camera are now automatically saved to the device's media library
- Edit menu for existing photos now uses a bottom sheet modal instead of inline buttons, providing better UX

**Date picker refinements**
- Date picker now displays "Today" when selecting the current date, instead of showing the formatted date (e.g., "Jun 13")
- This improves clarity when adding new climbs with today's date while browsing past climbs

**Form experience**
- Climb form properly receives and displays pre-selected photos from the FAB
- New `RouteForm` accepts optional `initialPhotoUri` parameter for seamless photo integration

## Under the hood

- Enhanced `PhotoPickerField` with:
  - Modal-based edit menu for better layout control
  - Automatic media library integration using `expo-media-library`
  - Fixed touch event handling with `pointerEvents="box-none"` for proper modal dismissal
- Updated `FloatingAddButton` to save photos to device library and pass them to the form
- Added `expo-media-library` plugin to app configuration
