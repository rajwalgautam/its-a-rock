# v0.4.2

A patch release focused on improving permissions and fixing location field behavior.

## What's fixed

**Location field behavior**
- Location field no longer auto-repopulates when you backspace to clear it, allowing you to enter a new gym
- Fix ensures the last location is suggested only on initial form load, not on every empty state

**Permissions**
- Removed invasive media library write permission from camera photo capture
- Camera now only requires camera permission (no additional photo library access needed)
- Photos from camera are stored in app cache and persist across sessions

## What's improved

**Status indicators**
- Active (uncompleted) climbs now display with a green status dot, making your active projects stand out
- Completed climbs show with an orange status dot to distinguish past accomplishments
- This visual hierarchy makes it easier to focus on what you're currently projecting

## Under the hood

- Enhanced `LocationPickerField` to use a ref-based approach for single-time initialization, preventing unwanted auto-population on backspace
- Updated photo handling in `FloatingAddButton` and `PhotoPickerField` to use camera URIs directly without media library dependency
- Removed `expo-media-library` imports from photo picker components
- Updated `RouteTile` and `RouteCard` status color logic to reflect active/completed states clearly
