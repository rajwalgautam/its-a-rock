# v1.1.0

This release lets every climb hold a full gallery of photos **and** videos, and
brings release notes into the app — a popup after you update, plus a browsable
history in Settings.

## What's new

**Photos & videos**
- **Multiple media per climb** — a climb is no longer limited to a single photo.
  Add as many photos and videos as you like to document progress and show off a
  send.
- **Gallery editor** — in the add/edit form you can add from your library or
  camera (including recording video), remove items, reorder them, and pick which
  photo is the cover shown on tiles.
- **Swipeable viewer** — tapping a climb's media opens a full-screen gallery you
  can swipe through; videos play in-app.
- Your existing climbs carry over automatically — each previous photo becomes the
  first item in that climb's new gallery, with nothing lost.

**Release notes in the app**
- **"What's new" popup** — after the app updates, it shows that version's notes
  once on first launch, so you can see what changed without hunting for them.
- **Release notes in Settings** — a new *Settings → About → Release notes* screen
  lists the last 10 releases with dates; expand any one to read its notes, or tap
  through to the full history on GitHub.

**Small touches**
- The location field's hint now reads "Add location here" so it's clearer what to
  type when logging a climb.

## Under the hood

- New `route_media` table (one-to-many, `ON DELETE CASCADE`) added via a
  `PRAGMA user_version` migration that backfills each existing `photo_uri` into
  position 0. `routes.photo_uri` is kept as a cached cover (the first **photo** in
  gallery order) so tiles/cards render without loading the whole gallery; detail
  loads populate `RouteWithGym.media` (#91).
- `RouteInput.media` is authoritative on write; when it's omitted (e.g. a
  long-press completion toggle on a list route, whose gallery isn't loaded) the
  media table is left untouched so quick edits never drop attachments.
  `MediaGalleryField` + the pure reducers in `mediaUtils` (add/dedupe, remove,
  move, set-cover) drive the editor; `MediaViewer` uses `expo-video` for in-app
  playback (#91).
- `WhatsNewGate` compares a persisted `lastSeenVersion` against
  `Constants.expoConfig.version` on launch, fetches the raw
  `changelogs/v<version>.md` (truncated at `## Under the hood`), and marks the
  version seen even on fetch failure so it never nags; fresh installs show
  nothing (#92).
- `app/releases.tsx` lists the 10 most recent releases from the GitHub releases
  API and lazily fetches each changelog on expand, reusing one tiny in-house
  `Markdown` renderer (no new markdown dependency) shared with the popup; a
  footer links out to all releases (#93).
- `stripUnderTheHood` and `shouldShowWhatsNew` are pure and unit-tested; new
  tests also cover the gallery reducers and asset→media mapping.
