# v1.2.2

A round of polish: saving a location takes one tap again, videos remember how you like them, and the grid goes all the way down to a single big tile.

## What's new

**Locations**
- **One tap to save** — adding or renaming a location now saves on the first tap. The dialog lifts above the keyboard so the **Save** button is never covered, and pressing the keyboard's return key saves too.

**Photos & videos**
- **Mute videos by default** — videos already opened muted; now there's a **Settings → Logging** toggle to change that if you'd rather they play with sound.
- **Tidier "Nice send!" prompt** — the "add a video of your send?" popup is now a centered, themed dialog matching the rest of the app instead of a system alert.

**Grid**
- **Column size dropdown** — the History tab's column control is now a dropdown, with a new **Single** option for one large tile per row alongside Large, Medium, and Small.

**Other**
- **Cleaner release notes** — changelogs no longer render with big stretches of empty space between sections.

## Under the hood

- `GymFormModal` wraps its centered dialog in a `KeyboardAvoidingView` (mirroring `RouteForm`) and the name field gains `returnKeyType="done"` + `onSubmitEditing`.
- New app-wide themed confirm dialog: `useConfirmStore` (promise-based imperative API) + `ConfirmModalHost` mounted at the root layout; `confirmAddVideo` now resolves through it, so the prompt survives the context menu unmounting.
- `muteVideosByDefault` added to `Settings` (default `true`), threaded through `useSettingsStore` and read by `MediaViewer`'s `VideoPage`.
- `ColumnDensity` widened to `1 | 2 | 3 | 4`; `ColumnDensityControl` reworked from a segmented control into a dropdown. `RouteGrid` already guards `columnWrapperStyle` for the single-column case.
- `Markdown` trims its source and collapses runs of blank lines so consecutive newlines render as a single paragraph break.
