# v0.3.0

The first release of **It's A Rock** — an offline-first bouldering logbook for
Android. This note covers everything built across the 0.1.0–0.3.0 milestones.

## What's new

**Log your climbs**
- Add a boulder problem with a photo (from your library or camera), a V-scale
  grade with optional `+`/`-`, the gym or location, start/send dates, notes, and
  whether you've sent it or are still projecting it. Every field except the gym
  is optional.

**My Climbing**
- A this-week summary at a glance: visits, sends, and active projects.
- A photo-first grid of your active projects, with a big `+` button to log a new
  climb.

**History**
- Browse every climb — sent and in-progress — in the same photo grid.
- Switch tile size between 2, 3, and 4 columns; the choice is shared with My
  Climbing and remembered.

**Tap and long-press**
- Tap a tile to open the climb, where you can view every detail and edit it
  in place — changes are saved and survive restarts.
- Long-press a tile for quick actions: flip between sent and project, or delete.

**Light & dark**
- Choose Light, Dark, or System in Settings. Your choice is remembered.

**Stays on your phone**
- No account, no servers. Everything lives in on-device storage and works fully
  offline.

**In-app updates**
- The app checks for new releases on startup and from Settings → About, and can
  download and install the update for you.

## Under the hood

Not shown in the published release notes.

- **0.1.0 — Foundation.** Expo SDK 55 / RN 0.83 / strict TypeScript scaffold with
  the `@/` alias; light/dark theme system (`ThemeProvider` + two palettes);
  two-project Jest harness (`unit` ts-jest/node, `ui` jest-expo/jsdom) with native
  mocks; SQLite schema (`gyms` + `routes`, WAL, indexes) and a typed query layer
  with gym dedupe; `useRouteStore` / `useSettingsStore` / `useUpdateStore`.
- **0.2.0 — Core app.** Shared UI (RouteTile, RouteGrid, ColumnDensityControl,
  GradePicker, PhotoPickerField, RouteForm, RouteCard, StatCard,
  FloatingAddButton) and all screens (3-tab layout, My Climbing, History,
  Settings, add + detail). Pure logic is unit-tested: `gradeUtils`, `routeStats`
  (Monday-based week math), `validators`, `formatters`, `dateUtils`,
  `gymUtils`. Date fields use lightweight set-today/clear controls (no native
  date-picker dependency).
- **0.3.0 — Release & distribution.** Four Android config plugins (compileSdk 36 /
  buildTools 35, material 1.12.0 pin, arm64-only + Gradle caching, env-based
  release signing); the `CI / Release` workflow ports water-tracker's, publishing
  the signed APK as a Release **in this repo** via the built-in `GITHUB_TOKEN`
  (no cross-repo PAT) and adding a typecheck gate; in-app update mechanism
  (`versionCompare`, `updateChecker` pointed at `rajwalgautam/its-a-rock`,
  `UpdateBanner`); `scripts/deploy-pixel.sh`; and `docs/releasing.md` +
  `docs/testing.md`.

Verified: `npm run typecheck` clean; `npm test` 42 passed / 1 skipped across both
projects; `npx expo config` resolves all plugins.
